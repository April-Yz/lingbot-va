# V1.2

## LingBot Action-Only DSRL Baseline

This version adds a minimal `lingbot_action_only_dsrl` baseline inside `lingbot-va`.

### What Changed

- LingBot now exposes a priority-1 `initial_noise` steering interface through:
  - `VA_Server.sample_future_latents(...)`
  - `VA_Server.sample_actions(..., initial_noise=...)`

- New local action-only DSRL package:
  - `wan_va/action_only_dsrl/modules.py`
  - `wan_va/action_only_dsrl/policy.py`
  - `wan_va/action_only_dsrl/robotwin_env.py`

- New training entry:
  - `script/run_lingbot_action_only_dsrl.py`

- New config:
  - `examples/embodiment/config/robotwin_lingbot_action_only_dsrl.yaml`

### Inputs

- formatted RoboTwin RGB observations
- `observation.state`
- language prompt

### Outputs

- LingBot action chunk
- steering diagnostics
- SAC metrics

### Current Validation Result

- `use_dsrl=false`: passes in mock mode
- `use_dsrl=true`: passes in mock mode and injects steering noise with shape `[1, 30, 2, 16, 1]`
- embodied-SAC metrics: visible in mock mode
- RoboTwin online training: still blocked by the current `pytorch3d` / Blackwell toolchain mismatch
