# LingBot-VA V1.1 Workspace Compatibility Update

## What Changed

- The attention module import path was hardened so `flash-attn` is optional at import time.
- Runtime now raises a clear error only when `attn_mode='flashattn'` is explicitly requested without a compatible `flash-attn` build.

## Why

In this workspace, the available `flash-attn` wheel is ABI-incompatible with the required `torch 2.9.0+cu126` stack. The upstream code imported `flash-attn` eagerly, which prevented even `torch` attention mode from working.

## Affected Modules

- `wan_va/modules/model.py`

## Expected Behavior

- Inference or evaluation configured with `attn_mode='torch'` can import and run without `flash-attn`.
- Configurations that request `attn_mode='flashattn'` fail fast with an explicit compatibility error.

## Usage Notes

- Keep `transformer/config.json` aligned with the intended mode:
  - training: `flex`
  - inference: `torch` or `flashattn`
- In this workspace, prefer `torch` unless a source-built `flash-attn` matching the current PyTorch/CUDA ABI is installed.
